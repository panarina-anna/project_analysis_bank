# Анализ A/B-теста и прогнозирование посещаемости банковского приложения

**Проект по оценке эффективности нового интерфейса мобильного приложения банка «Деньги для Вас»**

## Описание проекта

Банк «Деньги для Вас» провёл A/B-тест для оценки нового улучшенного интерфейса мобильного приложения.  
**Цель эксперимента** — повысить активность и вовлечённость клиентов.  

**Группы теста:**
- **Группа A (контрольная)** — старый интерфейс  
- **Группа B (тестовая)** — новый интерфейс с упрощённой навигацией и персональными советами  

**Задачи проекта:**
1. Оценить успешность A/B-теста
2. Построить прогнозную модель посещаемости приложения
3. Визуализировать результаты
4. Подготовить выводы и рекомендации для бизнеса

## Этапы работы

### 1. Предобработка данных
- Приведение названий столбцов к `snake_case`
- Приведение типов данных (`datetime`, `category`)
- Обработка пропусков:
  - Возраст — заполнен медианой по группам
  - Город — заполнен `'unknown'`
  - Логические пропуски в событиях (`screen_name`, `button_name`, `transaction_type`) — удалены или заменены на `'no_...'`
- Удаление дубликатов (явных и неявных)
- Обработка выбросов:
  - Удалены пользователи с возрастом < 14 или > 100
  - Проверка логики событий (нет `logout` без `login`)
- Объединение редких городов в категорию `'Другой'`

### 2. Расчёт метрик A/B-теста
- Выделение сессий на основе событий `login` / `logout`
- Для каждого пользователя рассчитаны:
  - Количество сессий
  - Среднее число событий за сессию
  - Конверсия в транзакцию
  - Средняя длительность сессии
- Агрегация метрик по группам A и B

### 3. Проверка статистических гипотез
**Гипотезы:**
| Гипотеза | Результат |
|----------|-----------|
| H1: Конверсия в группе B выше | не подтвердилась |
| H2: Количество сессий в группе B отличается | подтвердилась |
| H3: Событий на сессию в группе B больше | не подтвердилась |
| H4: Длительность сессии в группе B больше | подтвердилась |

**Методология:**
- Проверка нормальности распределений (тест Шапиро-Уилка)
- Выбор критерия: t-тест или U-тест Манна-Уитни
- Уровень значимости α = 0.05

### 4. Регрессионное моделирование
**Цель:** прогнозирование ежедневной посещаемости приложения

**Созданные признаки:**
- День недели, месяц, выходной/праздник
- Лаговые значения (1, 2, 7, 14 дней)
- Скользящие средние (7, 14 дней)

**Сравнение моделей:**
| Модель | MAE (посещений) | MAPE | RMSE |
|--------|-----------------|------|------|
| Linear Regression | 1213 | 10.7% | 1798 |
| Random Forest | **951** | **8.1%** | **1433** |

**Вывод:** Random Forest показывает лучшую точность прогноза

## 📊 Ключевые результаты

### A/B-тест
- **Конверсия:** 75.4% (A) vs 76.2% (B) — разница статистически не значима (p = 0.236)
- **Количество сессий:** +31.9% в группе B (p < 0.001)
- **Длительность сессии:** +93.3% в группе B (p < 0.001)
- **События на сессию:** разница не значима (p = 0.055)

### Прогнозная модель
- Лучшая модель: **Random Forest**
- MAPE: **8.1%** — модель ошибается в среднем на 8% от реальных значений
- Топ-3 признака: лаг 1 дня, выходной день, день недели

## Выводы и рекомендации

**Выводы:**
1. Новый интерфейс **не повлиял на конверсию** (она осталась на высоком уровне около 99%)
2. Новый интерфейс **значимо увеличил вовлечённость**:
   - Пользователи заходят чаще (+32%)
   - Проводят в приложении почти в 2 раза больше времени
3. Прогнозная модель Random Forest позволяет с высокой точностью предсказывать нагрузку на приложение (MAPE 8.1%)

**Рекомендации:**
1. **Внедрить новый интерфейс для всех пользователей** — он доказал свою эффективность в увеличении вовлечённости
2. **Использовать прогнозную модель** для планирования нагрузки на серверы и маркетинговых активностей
3. **Провести A/A-тест** перед следующими экспериментами для проверки корректности сплитования трафика
4. **Проанализировать долгосрочный эффект** через 1–2 месяца после полного внедрения
